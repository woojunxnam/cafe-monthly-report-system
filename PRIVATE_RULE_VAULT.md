# Private Rule Vault

## 왜 public repo에 규칙을 넣지 않는가

계좌별 비공개 규칙에는 실제 운영 매핑, 관련자금 그룹, 제외 기준 같은 민감 정보가 들어갑니다.  
이 정보는 public 저장소에 커밋하면 안 되므로, 공개 저장소에는 구조와 예시만 두고 실제 규칙은 로컬에서만 사용합니다.

## 로컬 금고 동작 방식

브라우저 로컬 비밀 규칙 금고는 사용자가 한 번 업로드한 계좌별 비공개 규칙 JSON을 브라우저 저장소에 보관합니다.

동작 순서:

1. 사용자가 계좌 카드에서 비공개 규칙 JSON을 업로드합니다.
2. `금고에 저장`을 누르면 해당 규칙이 계좌 slot 기준으로 저장됩니다.
3. 다음 방문 시 저장된 규칙이 자동 로드됩니다.
4. 사용자는 거래 파일만 다시 업로드해도 리포트를 생성할 수 있습니다.

## 저장 위치

1차 저장소:

- IndexedDB

fallback:

- localStorage

현재 구현 파일:

- [privateRuleVault.js](/c:/Users/남우현/Desktop/cafe_monthly_report_final_pack/src/core/privateRuleVault.js)

저장 항목:

- `accountId`
- `rules`
- `updatedAt`
- `schemaVersion`

## export / import 방식

내보내기:

- 계좌별 저장 규칙을 JSON 파일로 저장합니다.
- 파일에는 `accountId`, `updatedAt`, `rules`, `exportVersion`이 포함됩니다.

가져오기:

- 저장해 둔 JSON을 다시 금고로 불러옵니다.
- 현재 계좌 slot과 다른 `accountId`가 들어 있으면 가져오기를 막습니다.

## 삭제 방식

- 계좌 카드의 `금고에서 삭제` 버튼으로 해당 slot 규칙만 삭제합니다.
- 금고 저장본만 삭제되며, 현재 세션에서 업로드한 임시 규칙은 별도로 남을 수 있습니다.

## 자동 로드 방식

- 페이지 초기 로드 시 account manifest를 순회합니다.
- 계좌별 금고 저장본이 있으면 자동으로 메모리에 적재합니다.
- UI에는 `자동 로드됨`, `저장됨`, `마지막 업데이트`를 표시합니다.
- 이후 리포트 생성 시 업로드 규칙이 없더라도 자동 로드된 금고 규칙이 override로 사용됩니다.

## 보안 주의사항

- 이 금고는 public 저장소가 아니라 사용자 브라우저 저장소를 사용합니다.
- 즉, GitHub Pages에 민감 규칙이 포함되지는 않습니다.
- 다만 같은 브라우저 프로필을 사용하는 사람은 해당 저장본에 접근할 수 있으므로 공용 PC에서는 사용 후 삭제가 필요합니다.
- 1차 구현에는 passphrase 암호화를 넣지 않았습니다.
- 더 강한 보호가 필요하면 export 후 별도 암호화 보관을 권장합니다.

## 향후 확장 포인트

2단계에서는 private GitHub repo를 백업 저장소로 붙일 수 있습니다.

확장 방향:

- 브라우저 로컬 금고를 1차 저장소로 유지
- 선택적으로 private GitHub repo에 암호화 백업 저장
- 로컬 우선, 원격 백업 보조 구조로 확장

이를 위해 현재 저장소 접근 로직은 UI와 분리된 모듈로 유지합니다.
