# Project Handoff

이 패키지는 “계좌 월간 리포트 자동화”의 현재 최신 상태를 파일로 고정한 것이다.

## 목적
- 대화 없이도 AI 에이전트가 현재 규칙을 이해하고 구현을 시작할 수 있게 한다.
- 월별 원본 파일을 넣고 같은 구조의 리포트를 반복 생성할 수 있게 한다.

## 지금 바로 봐야 할 것
1. `CURRENT_DECISIONS.md`
2. `config/category_map.yaml`
3. `config/deposit_rules.yaml`
4. `output/2026-03/report_2026-03.html`

## 구현 우선순위
1. 원본 은행파일 읽기 로직 고정
2. 출금/입금 바인딩 함수 고정
3. HTML 템플릿 고정
4. 월별 자동 실행 스크립트 작성
5. 이후 GitHub Pages 배포

## 주의
- 다른 계좌를 합치기 전에 계좌별 매핑 차이를 먼저 검토
- ATM입금은 반드시 별도 카테고리 유지
- 박재민/성기용은 운영지출에서 제외하되 관련자금 섹션에서 반드시 유지
